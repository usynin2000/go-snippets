## Что такое Context?

> **Context (контекст)** — это стандартный механизм в Go для управления временем жизни операций, передачи сигналов отмены и передачи данных между границами API и горутинами.

## Основные функции Context:

1. **Отмена операций** — позволяет отменить длительные операции
2. **Таймауты и дедлайны** — автоматическая отмена по истечении времени
3. **Передача значений** — передача request-scoped данных (например, user ID, request ID)

## Основные типы:

- `context.Background()` — корневой контекст, никогда не отменяется
- `context.TODO()` — используется когда не уверены какой контекст использовать
- `context.WithCancel()` — контекст с возможностью отмены
- `context.WithTimeout()` — контекст с таймаутом
- `context.WithDeadline()` — контекст с конкретным временем отмены
- `context.WithValue()` — контекст с переданными значениями


## Правила использования:

1. Context передается как первый параметр функции
2. Не храните контекст в структурах
3. Никогда не передавайте nil контекст (используйте context.TODO())
4. WithValue используйте только для request-scoped данных
5. Всегда вызывайте cancel() функцию для освобождения ресурсов


## Где используется:

- HTTP серверы и клиенты
- Запросы к базам данных
- gRPC вызовы
- Управление горутинами
- Любые долгие операции, которые могут быть отменены
